## Overview

**Divya.ai** is a full-stack personal portfolio and intelligent resume generation platform built for AI/ML engineers and healthcare AI researchers. It combines a premium warm ivory editorial aesthetic with live GitHub integration, an AI-powered admin panel, and a global resume engine that automatically detects the visitor's country and generates the correct regional PDF format вЂ” ATS-clean for western markets, photo-included with personal details for East Asia and Europe.

The entire platform runs from a single source of truth вЂ” `data.json`. Whatever is live on the website is exactly what appears in the resume. No manual resume editing. No stale data. No separate resume file to maintain.

> This project itself is managed and showcased through the portfolio it powers вЂ” the resume you are reading about was generated by the system itself.

## Key Features

### рџЊЌ Global Resume Engine
Detects visitor's country from IP address using `ip-api.com`. Automatically generates the correct regional PDF вЂ” no dropdowns, no manual selection. Template A is ATS-optimised single column with no photo for western markets. Template B includes photo, date of birth, and regional personal fields for East Asia, Europe, and Middle East. Both templates render from the same `data.json` via Jinja2 + Playwright.

### рџ”ґ Live GitHub Integration
Projects are fetched in real time from GitHub REST API. Forked repos are filtered out. Groq API (Llama models) generates a one to two sentence summary per repo on first fetch and caches it permanently. A GitHub webhook invalidates the cache the moment a new push lands. Contribution heatmap rendered from GitHub GraphQL API as an SVG grid.

### рџ¤– AI Admin Panel
Accessible only at `/admin` вЂ” no link exists anywhere on the visible site. Contains 19 management sections including a natural language command center powered by Groq API. Type `hide the flask project` or `rewrite my bio` and the system executes the action directly. All changes persist in `data.json` and reflect immediately on both the website and the next resume download.

### рџ“„ Dynamic Resume вЂ” Always Current
The resume PDF is generated fresh on every download. It reads `data.json` at request time, fetches live GitHub repos, and builds the PDF through Playwright. If any section in `data.json` is empty it is silently skipped. Nothing is invented. Nothing is hardcoded. Education is fully dynamic and supports future degrees without code changes.

### вњ¦ Antigravity Aesthetic
55 upward-floating canvas particles fixed behind all content вЂ” particles always drift upward, never downward. This is the core antigravity metaphor. All cards, skill pills, and floating elements use Framer Motion `animate={{ y: [0, -12, 0] }}` with varied durations and delays so nothing bobs in sync. Three.js wireframe geometry rotates in the hero background. Custom cursor with 8px dot and 32px trailing ring.

### рџ“Љ Visitor Analytics
Tracks page views, resume downloads (with hashed IP and detected country), project clicks, and contact form submissions. All events stored in `data.json` analytics array. Viewable in the admin analytics dashboard with Recharts bar charts.

## Installation

### Prerequisites
- Node.js v18+
- Python 3.10+
- Groq API key from [console.groq.com](https://console.groq.com)
- GitHub Personal Access Token (for GraphQL heatmap)

### Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
playwright install chromium
cp .env.example .env            # fill in your API keys
uvicorn main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

## Academic & Research Context

This project was designed with a dual audience in mind вЂ” technical recruiters evaluating full-stack capability, and academic researchers evaluating systems thinking and AI integration depth.

**For recruiters:** The system demonstrates production-grade patterns including async API design, webhook-driven cache invalidation, region-aware PDF generation, and AI pipeline integration.

**For researchers:** The architecture reflects a data-flow-first philosophy вЂ” a single data source propagates through a rendering layer, an AI enrichment layer, and a localisation layer to produce multiple output artefacts (website, resume, analytics). The resume generation pipeline is deterministic and reproducible вЂ” the same `data.json` always produces the same PDF output.
